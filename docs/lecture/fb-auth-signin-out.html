# Firebase 인증 (로그인/로그아웃)

# Google 로그인

Firebase 인증(Authentication) 서비스를 사용해 Google 로그인, 로그아웃 하는 방법을 학습합니다.

# Google 로그인 사용 설정

Authentication → Sign-in method 패널에서 "Google 로그인 제공업체"를 선택해 사용 설정 합니다.

# GoogleAuthProvider 설정

Google 로그인 제공업체(GoogleAuthProvider) 객체를 생성한 후, 외부에서 호출해 사용할 수 있도록 설정합니다.




 
 


 

// ...

// Google 인증 프로바이더
const googleProvider = new firebase.auth.GoogleAuthProvider()
googleProvider.setCustomParameters({ prompt: "select_account" })

// 로그인 함수 내보내기
export const signInWithGoogle = () => auth.signInWithPopup(googleProvider)

setCustomParameters() 메서드

팝업 및 리디렉션 로그인 작업에 대한 Google OAuth 요청을 전달할 OAuth 맞춤 매개 변수를 설정합니다. 유효한 매개 변수는 'hd', 'hl', 'include_granted_scopes', 'login_hint' 및 'prompt' 입니다.

googleProvider.setCustomParameters({ prompt: 'select_account' })
1

사용자 권한을 부여하는 서버는 사용자에게 사용자 계정을 선택하라는 메시지를 표시합니다. 이를 통해 여러 계정을 가진 사용자는 현재 세션이 있을 수 있는 여러 계정 중에서 하나를 선택할 수 있습니다. 따라서 사용자 컴퓨터에 등록된 여러 Google 계정 중 하나를 선택할 수 있도록 하려면 이 매개 변수를 포함해야합니다.

자세한 내용은 firebase.auth.GoogleAuthProvider 공식 문서 (opens new window)를 참고합니다.

# Google 로그인/로그아웃 함수

앱에서 사용 할 Google 로그인(signIn), 로그아웃(signOut) 함수를 작성합니다.

 





 
 
 
 
 
 


 
 
 



 
 




import { auth, signInWithGoogle, signOut as signOutGoggle } from "./services/firebase"


export default function App() {
  
  // Google 로그인 함수
  const signIn = () => {
    signInWithGoogle()
      .then((response) => response.user)
      .then((user) => console.log(`${user.displayName}님 로그인 인증에 성공했습니다.`))
      .catch((error) => console.error(error.message));
  };
  
  // Google 로그아웃 함수
  const signOut = () => {
    signOutGoggle().then(() => console.log("로그아웃"));
  };

  return (
    <div className="App">
      <button type="button" onClick={signIn}>Google 로그인</button>
      <button type="button" onClick={signOut}>로그아웃</button>
    </div>
  );
}

# 로그인(인증) 감지 이벤트

로그인 사용자 상태를 추가하고, 로그인 여부를 감지하는 이벤트 onAuthStateChanged (opens new window) 메서드를 작성합니다.








 
 
 
 
 

























export default function App() {
  // 로그인(인증) 사용자 상태
  const [currentUser, setCurrentUser] = React.useState(null)
  
  // 사이드 이펙트
  React.useEffect(() => {
    // 로그인 여부 감지 이벤트
    auth.onAuthStateChanged((currentUser) => {
      currentUser ?
        setCurrentUser(currentUser) :
        setCurrentUser(null)
    })
  }, [])

  // ...

  return (
    <div className="App">
      <img src={googlePath} alt="" height="80" />
      <p>
        {currentUser
          ? currentUser.displayName
          : "서비스를 이용하려면 로그인 해야 합니다."}
      </p>
      {!currentUser ? (
        <button type="button" onClick={signIn}>
          Google 로그인
        </button>
      ) : (
        <button type="button" onClick={signOut}>
          로그아웃
        </button>
      )}
    </div>
  )
}

# 로그인(인증) 감지 이벤트 구독 취소

onAuthStateChanged (opens new window) 메서드는 구독(subscribe)을 취소할 수 있는 구독 취소(unsubscribe) 메서드를 반환합니다. 이를 사용해 앱이 언마운트 될 때 로그인(인증) 감지 이벤트를 구독 취소할 수 있도록 실습을 진행합니다.



 
 
 
 
 



 



React.useEffect(() => {
  // 구독 취소 함수 참조
  const unsubscribe = auth.onAuthStateChanged((currentUser) => {
    currentUser ? 
      setCurrentUser(currentUser) : 
      setCurrentUser(null)
  })

  // [클린업 함수] 구독 취소 
  return () => {
    unsubscribe()
  }
}, [])

클래스 컴포넌트의 구독 취소(unsubscribe)

클래스 컴포넌트의 라이프 사이클 메서드를 사용해 로그인(인증) 감지 이벤트를 컴포넌트가 언마운트 되기 전에 구독 취소 해야 합니다.



 


 
 
 
 
 



 



class App extends React.Component {
  // ...
  unsubscribe = null

  componentDidMount() {
    this.unsubscribe = auth.onAuthStateChanged((currentUser) => {
      currentUser ? 
        setCurrentUser(currentUser) :
        setCurrentUser(null)
    })
  }

  componentWillUnmount() {
    this.unsubscribe()
  }
}

# 인증 상태 유지 설정

일반적으로 로그인 상태는 창을 닫고 다시 열어도 유지되므로 사용자에게 매우 편리한 경험을 제공합니다. 하지만 민감한 정보를 가진 서비스 또는 다른 이들과 함께 사용하는 서비스일 경우 세션이 만료되거나, 창을 닫았을 때 바로 로그아웃 되어서비스를 이용할 수 없도록 만들어야 합니다.








 

import firebase from 'firebase/app'
import 'firebase/auth'

export const auth = firebase.auth()

// 로그인(인증) 상태 유지 설정
// - 열린 세션(창)에서만 로그인 상태 유지
auth.setPersistence(firebase.auth.Auth.Persistence.SESSION)

Firebase 서비스는 상황에 따라 앱에서 인증 상태를 유지 또는 만료 시킬 수 있는 옵션을 제공합니다. 이 옵션을 사용해 인증 상태를 유지 또는 열린 세션에서만 유지, 창을 닫으면 인증 만료를 선택하여 처리할 수 있습니다

상태 값 설명
firebase.auth.Auth.Persistence.LOCAL 'local' 창을 닫아도 로그인 상태 유지 (기본 값)
firebase.auth.Auth.Persistence.SESSION 'session' 열린 세션(창)에서만, 로그인 상태 유지
firebase.auth.Auth.Persistence.NONE 'none' 창을 닫으면 로그오프

보다 자세한 사용법은 인증 상태 지속성 공식 문서 (opens new window)를 참고하세요.
