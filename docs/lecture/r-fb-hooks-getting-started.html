# 소개

React Firebase Hooks (opens new window)은 Firebase (opens new window) 서비스 활용을 위한 훅(Hooks) 라이브러리입니다.

# 훅을 사용하는 이유

React Hooks은 좀 과장된 면이 없잖아 있지만, 훅이 React 앱 개발에 편의성을 높여준 것은 사실입니다. 훅을 사용한 방법이 이전에 사용되던 방법 보다 훨씬 간단합니다. 이 라이브러리는 RxFire (opens new window)에서 영감을 얻었으며, React Hooks 출시 전에 여러 앱에서 사용되던 라이브러리를 기반으로 합니다.

# 라이브러리 훅

React Firebase Hooks 라이브러리가 제공하는 훅은 다음의 4가지입니다.

  • Auth Hooks
  • Realtime Database Hooks (준비중)
  • Cloud Firestore Hooks (준비중)
  • Cloud Storage Hooks (준비중)

React Native 0.59.0 이상 버전부터 훅을 공식 지원합니다. React Firebase Hooks은 Firebase JS SDK 및 React Native Firebase와 함께 작동하지만, 일부 Flow 및 Typescript 유형(types)이 문제가 되는 경우가 있습니다. React Native Firebase 사용자를 위해 이를 개선하는 방법을 찾고 있습니다.

# 설치

firebase (opens new window) JS 클라이언트를 설치합니다.

$ npm i firebase
1

react-firebase-hooks (opens new window) 패키지를 프로젝트에 설치합니다.

$ npm i react-firebase-hooks
1

React Firebase Hooks은 React 16.8.0 이상, Firebase v5.0.0 이상에서 정상 작동합니다.

# Firebase 구성 정보

아래 URL의 프로젝트를 프로젝트 이름으로 변경해 브라우저 주소 창에 입력하면 Firebase 구성 정보를 확인할 수 있습니다.

https://console.firebase.google.com/project/프로젝트/settings/general

const firebaseConfig = {

# Firebase 앱 초기화

firebase/config.js 파일에 다음과 같이 작성하여 Firebase 앱을 초기화 하도록 구성합니다.

// Firebase 앱 패키지 호출
import * as firebase from "firebase/app";

// Firebase 앱 KEY & ID
const {
  REACT_APP_FB_API_KEY,
  REACT_APP_FB_PROJECT_ID,
  REACT_APP_FB_MESSANGER_SENDER_ID
} = process.env;

// Firebase 앱 구성
export const config = {
  apiKey: REACT_APP_FB_API_KEY,
  projectId: REACT_APP_FB_PROJECT_ID,
  messagingSenderId: REACT_APP_FB_MESSANGER_SENDER_ID,
  authDomain: "인증-도메인-이름",
  databaseURL: "데이터베이스-이름-URL",
  storageBucket: "스토리지-버킷",
};

// Firebase 앱 초기화
firebase.initializeApp(config);
