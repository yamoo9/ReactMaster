# Formik 컴포넌트

Formik 라이브러리가 제공하는 컴포넌트를 사용해 폼 컨트롤 실습을 진행합니다.

  • <Formik /> (opens new window)
  • <Form /> (opens new window)
  • <Field /> (opens new window)
  • <ErrorMessage /> (opens new window)
  • <FieldArray /> (opens new window)
  • <FastField /> (opens new window)

# 실습

"간단한 폼 컨트롤" 실습 과정에 이어 진행합니다.

# Formik 컴포넌트

<Formik /> 컴포넌트를 formik 패키지로부터 불러옵니다.

import { Formik } from 'formik'
1

useFormik 훅을 제거한 후, <form> 요소를 감싸도록 <Formik /> 컴포넌트를 추가한 다음 다음과 같이 초깃값, 유효성 검사 스키마, 전송 이벤트 리스너 속성에 이전 설정 값을 연결합니다.


 
 
 




<Formik
  initialValues={initialValues}
  validationSchema={validationSchema}
  onSubmit={onSubmit}
>
  {/* .... */}
</Formik>

# Field 컴포넌트

<Field /> 컴포넌트를 formik 패키지로부터 불러옵니다.

import { Formik, Form, Field } from 'formik'
1

<input /> 요소를 <Field /> 컴포넌트로 변경하고 getFieldProps 전개 코드를 제거합니다.









 
 
 
 
 









<Formik
  initialValues={initialValues}
  validationSchema={validationSchema}
  onSubmit={onSubmit}
>
  <Form>
     <div className="formControl">
      <label htmlFor="userName">이름</label>
      <Field
        type="text"
        id="userName"
        name="name"
      />
      {formik.touched.name && formik.errors.name && (
        <span role="alert">{formik.errors.name}</span>
      )}
    </div>

    {/* .... */}
  </Form>
</Formik>

폰 번호를 검사할 정규 표현식이 매칭되도록 설정 (opens new window)하고, 올바르지 않을 경우 표시할 오류 메시지도 추가합니다.

정규 표현식 설명
/^\d{2,3}-\d{3,4}-\d{4}$/ 전화번호 양식 (예: 02-123-1234, 070-1234-1234)
/^\d{3}-\d{3,4}-\d{4}$/ 휴대폰 양식 (예: 010-123-1234, 010-1234-1234)




















 
 
 
 
 
 
 
 


const validationSchema = Yup.object({
  name: Yup.string()
    .required('이름` 입력은 필수입니다.')
    .min(2, '`이름`은 최소 2자 이상 입력되어야 합니다.')
    .trim(),
  email: Yup.string()
    .required('`이메일` 입력은 필수입니다.')
    .email('이메일 입력 형식이 올바르지 않습니다. 확인 후, 다시 입력하세요.'),
  channel: Yup.string()
    .required('채널` 입력은 필수입니다.')
    .min(2, '`채널`은 최소 2자 이상 입력되어야 합니다.')
    .trim(),
  interests: Yup.string().oneOf(['react', 'svelte', 'vue', 'angular']),
  comments: Yup.string()
    .max(300, '`코멘트`는 최대 300자까지 입력이 가능합니다.')
    .trim(),
  sns: Yup.object().shape({
    facebook: Yup.string().min(2, '2자 이상 입력해야 합니다.').trim(),
    github: Yup.string().min(2, '2자 이상 입력해야 합니다.').trim(),
  }),
  phones: Yup.array().of(
    Yup.string()
      .matches(
        /(^\d{3}-\d{3,4}-\d{4}$|^\d{2,3}-\d{3,4}-\d{4}$)/,
        '집 전화번호 또는 폰 번호 입력 양식이 잘못되었습니다.'
      )
      .trim()
  ),
})

# FieldArray 컴포넌트

<FieldArray /> (opens new window) 컴포넌트는 리스트(배열)을 관리할 때 사용합니다. 이 컴포넌트를 사용해 사용자로부터 하나 이상 전화 번호를 입력 받을 수 있도록 설정해봅니다. 하나 이상 배열 아이템을 입력 받을 수 있도록 multiPhones 상태를 추가합니다.












 


const initialValues = {
  name: '',
  email: '',
  channel: '',
  interests: '',
  comments: '',
  sns: {

# FastField 컴포넌트

<FastField /> (opens new window) 컴포넌트는 성능 최적화를 위한 필드 요소입니다. 최적화가 필요한 시점에 사용할 수 있습니다. <Field /> 컴포넌트는 사용자가 입력할 때마다 컴포넌트를 다시 렌더링합니다. 반면 <FastField />는 컴포넌트에 직접 입력할 때만 컴포넌트를 다시 렌더링 합니다.










 

 





























 






<div className="formControl">
  <label htmlFor="multiPhones">기타 폰 번호</label>
  <FieldArray
    name="multiPhones"
    render={({ push, remove, form }) => {
      const { multiPhones } = form.values
      return multiPhones.map((phone, index) => {
        const phoneId = `multiPhones[${index}]`
        return (
          <FastField key={`phone-${index}`} name={phoneId}>
            {({ field, form, meta }) => {
              console.log('`기타 폰 번호` 입력 필드에 입력할 때만 컴포넌트를 다시 렌더링 합니다.')
              return (
                <>
                  <div className="phoneControl">
                    <input
                      type="text"
                      id={index === 0 ? 'multiPhones' : null}
                      {...field}
                    />
                    <button
                      type="button"
                      aria-label="새 전화번호 추가"
                      onClick={() => push('')}
                    >
                      +
                    </button>
                    {multiPhones.length > 1 && (
                      <button
                        type="button"
                        aria-label="전화번호 제거"
                        onClick={() => remove(index)}
                      >
                        -
                      </button>
                    )}
                  </div>
                  <AlertMessage name={phoneId} />
                </>
              )
            }}
          </FastField>
        )
      })
    }}
  />
</div>

# 유효성 검사 컨트롤

유효성 검사가 실행되는 때는 onChange, onBlur 이벤트가 발생 (opens new window)하는 경우입니다. 매번 입력할 때 또는 블러 상태에서 유효성 검사를 수행하지 않도록 하려면 다음과 같이 validateOn* 값을 false로 설정합니다.





 
 




<Formik
  initialValues={initialValues}
  validationSchema={validationSchema}
  onSubmit={onSubmit}
  validateOnChange={false}
  // validateOnBlur={false}
>
  {/* ... */}
</Formik>

# 개별 필드 유효성 검사

유효성 검사는 일반적으로 일괄 관리하지만, 때론 개별적으로 필드마다 유효성 검사를 하고 싶을 수 있습니다.

작성한 유효성 검사 스키마 중, 개별 검사를 테스트 할 부분을 주석 처리합니다.














 
 
 






















const validationSchema = Yup.object({
  name: Yup.string()
    .required('이름` 입력은 필수입니다.')
    .min(2, '`이름`은 최소 2자 이상 입력되어야 합니다.')
    .trim(),
  email: Yup.string()
    .required('`이메일` 입력은 필수입니다.')
    .email('이메일 입력 형식이 올바르지 않습니다. 확인 후, 다시 입력하세요.'),
  channel: Yup.string()
    .required('채널` 입력은 필수입니다.')
    .min(2, '`채널`은 최소 2자 이상 입력되어야 합니다.')
    .trim(),
  interests: Yup.string().oneOf(['react', 'svelte', 'vue', 'angular']),
  // comments: Yup.string()
  //   .max(300, '`코멘트`는 최대 300자까지 입력이 가능합니다.')
  //   .trim(),
  sns: Yup.object().shape({
    facebook: Yup.string().min(2, '2자 이상 입력해야 합니다.').trim(),
    github: Yup.string().min(2, '2자 이상 입력해야 합니다.').trim(),

  phones: Yup.array().of(
