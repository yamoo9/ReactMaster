# Firestore (사용자 프로필 생성)

Firebase 데이터베이스(Cloud Firestore) 사용법을 학습합니다.

# Firestore 데이터베이스 설정

Cloud Firestore 계층 구조는 Collection → Document → Field로 구성됩니다.

# 컬렉션 생성

Cloud Firebase 데이터베이스 설정을 통해 사용자 정보를 저장 할 users 컬렉션(collection)을 추가합니다.

# 도큐멘트(문서) ID → 필드 생성

컬렉션은 "문서의 집합"으로 하위에 문서를 추가할 수 있습니다. 각 문서는 식별 가능한 고유 ID 값을 가져야 합니다. 그리고 문서의 데이터는 각 필드로 저장됩니다. Cloud Firestore에서 직접 데이터 필드를 추가할 수 있습니다.

문서(Document) ID

문서의 ID는 임의로 생성 가능하지만, "인증 사용자의 고유 ID(uid)" 값을 입력해 저장합니다.

# 데이터베이스 구조

컬렉션(collection), 문서(document), 필드(field) 구성이 완료되면 Cloud Firestore에 다음과 같이 표시됩니다.

작성한 Cloud Firestore 데이터베이스 구조를 JavaScript 객체로 표현하면 다음과 같이 작성할 수 있습니다.

{
  // [컬렉션] 인증 사용자
  "users": {
    // [도큐멘트] 사용자 UID
    "nkGxEOrO5VchpOUUNXcPjKrQkSp1": 
      // [필드] 사용자 정보
      {
        "displayName": "야무",
        "email": "yamoog@gmail.com",
        "emailVerified": true,
        "photoURL": "https://lh3.googleusercontent.com/a-/AOh14GhgrBSraw=s96-c"
      }
  }
}

# 임시 문서 제거

데이터베이스 구조를 이해하기 위해 직접 생성한 임시(dummy) 문서를 Users 컬렉션에서 제거합니다.

# 사용자 프로필 생성 유틸리티

Firebase 인증(auth) 사용자 정보를 전달 받아 Firestore 데이터베이스 Users 컬렉션에 새로운 문서를 저장하는 유틸리티 함수를 작성합니다.

Firestore의 Users 컬렉션에서 문서를 가져오거나, 추가하는 방법은 데이터 읽기 (opens new window), 데이터 추가 (opens new window) 공식 문서를 참고합니다.

// Firestore 데이터베이스에 인증 사용자 프로필 저장 유틸리티
export const createUserProfile = async (user, additionalData = {}) => {
  // user 값이 전달되지 않을 경우 오류 출력
  if (!user) {
    return new Error(
      "createUserProfile 유틸리티에 user 값이 전달되지 않았습니다."
    )
  }

  // Firestore Users 컬렉션로으부터 사용자(문서 ID) 참조
  // 아래 작성된 구문은 db.collection('Users').doc(user.uid) 와 동일합니다.
  const userRef = db.doc(`Users/${user.uid}`)
  // userRef 참조로부터 스냅샷 가져오기 (대기...)
  const snapshot = await userRef.get()

  // 스냅샷 exists 속성 확인 (존재 유무 파악)
  // 존재하지 않을 경우, 데이터베이스에 인증 사용자 프로필 저장
  if (!snapshot.exists) {
    try {
      // 저장할 사용자 정보 추출
      const { displayName, email, emailVerified, photoURL } = user
      // 생성 날짜
      const createdAt = new Date()

      // 데이터베이스에 새로운 인증 사용자 프로필 저장
      await userRef.set({
        displayName,
        email,
        emailVerified,
        photoURL,
        createdAt,
        ...additionalData
      })
    } catch (error) {
      // 오류 발생 시, 오류 메시지 출력
      console.error("", error.message)
    }
  }

  // 사용자 참조 반환
  return userRef
}

# currentUser 상태 업데이트

Firestore에 생성된 사용자 프로필을 가져와 컴포넌트의 currentUser 상태를 업데이트 하려면 다음과 같이 코드를 작성합니다.

Firestore의 Users 컬렉션에서 실시간 업데이트 가져오기 (opens new window) 공식 문서를 참고합니다.





 



 


 
 
 
 
 
 
 
 









const [currentUser, setCurrentUser] = React.useState(null)

React.useEffect(() => {

  const unsubscribe = auth.onAuthStateChanged(async (currentUser) => {
    // 인증 사용자 정보가 있다면
    if (currentUser) {
      // createUserProfile 유틸리티의 반환 값(사용자 참조, Promise 객체)
      const userRef = await createUserProfile(currentUser)

      // 사용자 참조의 스냅샷을 가져와
      userRef.onSnapshot((snapshot) => {
        // 컴포넌트 상태 업데이트
        setCurrentUser({
          // 스냅샷의 ID (=== 인증 사용자의 uid)
          id: snapshot.id,
          // 스냅샷 데이터 전개 (객체 합성)
          ...snapshot.data()
        })
      })
    } else {
      setCurrentUser(null)
    }
  })

  return () => unsubscribe()
}, [])
