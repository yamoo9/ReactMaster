# react-spring 애니메이션 라이브러리

react-spring (opens new window)은 React UI 애니메이션을 위한 스프링 물리(spring-physics) 기반 애니메이션 라이브러리 입니다. react-spring 라이브러리를 사용해 React 앱 UI에 활기를 손쉽게 불어넣을 수 있습니다.


이 라이브러리는 Christopher Chedeau의 animated (opens new window)와 Cheng Lou의 react-motion (opens new window)에서 영감을 받아 탄생했습니다. animated의 강력한 인터폴레이션(보간법), 성능 그리고 react-motion의 사용 편의성을 상속합니다. 영감을 제공한 animated는 명령형으로 작성하고, react-motion은 선언형으로 작성합니다. 반면, react-spring은 양쪽 방법으로 모두 작성 가능합니다. react-spring이 제공하는 가볍고 사용하기 쉬운 유틸리티 함수를 통해 UI를 구성하는 로직과 별개로 정적 데이터를 사용해 모션을 적용할 수 있습니다.

# 설치

react-spring (opens new window) 패키지를 NPM 명령을 사용해 설치합니다.

$ npm install react-spring
1

# 플랫폼

react-spring은 크로스 플랫폼을 지향하며 웹, react-native, react-native-web 및 다른 플랫폼을 거의 지원합니다. (이 경우 /universal 내보내기를 사용해야 하며, 네이티브 요소와 컬러 인터폴레이션은 지원하지 않음)

# 브라우저 호환

react-spring 라이브러리는 BrowserList >1%, not dead, not ie 11, not op_mini all (opens new window) 구성을 사용하여 ES 모듈로 컴파일 됩니다. (IE 미지원) 필요한 경우 .cjs를 추가하여 commonJS로 내보낼 수도 있습니다.

# 파일 크기

모든 것을 포함한 파일 크기는 현재 10.7KB(web), 9.7KB(react-native) 또는 6.9KB(/universal 내보내기) 입니다. 파일 크기는 빌드 체인(build-chain)에 따라 다르며 트리 쉐이킹(tree-shaking, 모듈 최적화 기법)에 의해 줄어들 수 있습니다. 예를 들어 컬러 인터폴레이션 없이 useSpring을 불러와 사용할 경우(/universal 내보내기) 4.7KB가 됩니다.

# 스프링?

작업할 원칙(principle)을 스프링(spring)이라고 합니다. 스프링에는 정의된 커브(curve)나 설정된 시간(duration)이 없습니다. 아마도 기존에 사용했던 애니메이션 API와 크게 달라 생소함을 느낄 수도 있을 겁니다. react-spring은 시간과 커브의 관점에서 애니메이션을 생각하지만, 그 자체로 스크린의 요소들을 자연스럽게 움직이려고 할 때 어려움에 직면했습니다. 현실 세계에선 그렇게 움직이는 것이 없기 때문입니다.

우리는 시간 기반 애니메이션에 너무 익숙해져 이 모든 것을 조화롭게 하는 것은 말할 것도 없고, 임의의 커브, 이징, 타임 워터폴(waterfalls, 폭포) 등을 다루면서 어렵게 구현하는 것을 당연하다고 생각했습니다. Andy Matuschak(Apple UI-Kit 개발자)는 "지속 시간과 커브로 매개 변수화(parameterized) 된 애니메이션 API는 기본적으로 지속적이고 유동적인 인터랙션과 반대됩니다" (opens new window)라고 예를 들어 표현했습니다.

스프링을 사용하면 애니메이션이 보다 쉽고 사용하기 편해집니다. 기본적으로 모든 것이 자연스러워 보이는 느낌이 들 것입니다. 자세한 내용은 아래 비디오를 시청해보세요.
