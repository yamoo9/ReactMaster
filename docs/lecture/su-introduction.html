# Semantic UI React 프레임워크

Semantic UI React (opens new window)는 Semantic UI (opens new window) 프레임워크를 React 애플리케이션에 통합하는 공식 프레임워크로 다음의 기능을 제공합니다.

  • jQuery → React로 재구성
  • 선언형 API
  • 컴포넌트 확장
  • 속기형 Props
  • 서브 컴포넌트
  • 상태 자동 제어

# jQuery → React로 재구성

jQuery (opens new window)는 인기있는 DOM 라이브러리로, 직관적이고 손쉬운 문법을 제공해 문서 객체에 접근하거나, 조작할 수 있습니다. 반면 React는 실제(Actual) DOM이 아닌, 가상(Virtual) DOM을 사용합니다. React는 선언된 상태가 업데이트 되면 자동으로 DOM의 변경 부분을 패치(Patch) 하지만, DOM 요소에 접근하거나, 조작하지는 않습니다.

실제 DOM 조작을 React의 가상 DOM과 동기화하여 유지하는 것은 불가능합니다. 그런 이유로 Semantic UI React는 Semantic UI의 jQuery 기능을 모두 React 앱에서 활용할 수 있도록 재구성 하였습니다.

# 컴포넌트 확장

Semantic UI React 컴포넌트가 렌더링 하는 HTML 요소(혹은 태그)를 변경하고자 한다면? 원하는 HTML 요소 또는 React 컴포넌트를 as 속성 값으로 전달합니다. 컴포넌트를 중첩, 추가하지 않고도 컴포넌트 기능 및 props를 활용할 수 있어 확장(Augmentation)은 매우 강력합니다.

# 렌더링 할 HTML 요소 변경

as 속성 값으로 원하는 HTML 요소 이름(string)을 전달해 렌더링 할 수 있습니다.

개발 코드

import { Header } from 'semantic-ui-react'

웹 표준, 접근성 문제

Semantic UI React의 <Image /> 컴포넌트는 필수 속성인 alt을 설정하지 않아 웹 표준, 접근성 모두 문제가 됩니다. 그러므로 위와 같이 사용하는 것은 권장하지 않습니다. 다음과 같이 사용하세요.

import { Label, Image } from 'semantic-ui-react'

<Label size="small">
  <Image src="veronika.jpg" alt="베로니카" />
</Label>

# 서브 컴포넌트

서브(하위) 컴포넌트를 사용하면 마크업에 완전한 접근이 가능합니다. 이와 같은 방법은 컴포넌트를 사용자 정의하는 유연성을 확보합니다.

import { Message, Icon } from 'semantic-ui-react'

<Message icon>
  <Icon name="circle notched" loading />
  {/* 서브 컴포넌트 Message.Content */}
  <Message.Content>
    {/* 서브 컴포넌트 Message.Header */}
    <Message.Header>잠깐!</Message.Header>
    내용을 가져오는 중입니다. 잠시만 대기해주세요.
  </Message.Content>
</Message>

컴파운드 컴포넌트

Semantic UI React 프레임워크에서 말하는 서브 컴포넌트는 "컴파운드(Compound, 혼합 된) 컴포넌트"로도 불립니다.

# 상태 자동 제어

React 세계관에는 제어 가능한(Controlled) 또는 제어 되지 않는(Uncontrolled) 컴포넌트 개념이 있습니다. 상태를 가지는(Stateful) 컴포넌트 자체적으로 상태를 관리합니다. 예를 들어 Dropdown 컴포넌트는 open 속성이 전달될 경우, onClick 이벤트 연결 없이 자동으로 클릭되어 열린 상태로 렌더링 됩니다. value는 onChange 이벤트 연결 없이 내부에서 처리됩니다.

import { Dropdown } from 'semantic-ui-react'

<Dropdown text="파일" open>
  <Dropdown.Menu>
    <Dropdown.Item text='새 파일 만들기' description="Ctrl + N" />
    <Dropdown text="파일 열기..." description="Ctrl + O" />
  </Dropdown.Menu>
</Dropdown>

만약 open 속성 또는 value 속성 값을 컴포넌트에 전달하면? Dropdown 컴포넌트는 전달 받은 props에 대한 컨트롤을 사용자가 전달한 값으로 처리합니다. 그 외 다른 props는 자동으로 제어 상태를 유지합니다. 다수의 제어 혹은 제어되지 않는 props를 혼합(Mix & Match)해 사용할 수 있습니다. props를 추가 또는 제거하여 언제든지 컴포넌트를 제어하여 작동시킵니다.

작동 원리가 궁금한가요?

Semantic UI React의 AutoControlledComponent (opens new window) 코드를 살펴보면 원리를 학습할 수 있을 것입니다.
