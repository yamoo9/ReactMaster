# 시작하기

React 앱 개발 과정에 TypeScript를 활용하는 방법을 살펴봅니다.

# TypeScript를 사용하는 이유

TypeScript를 사용해 React 앱을 개발하면 보다 뛰어난 개발자 경험을 얻을 수 있습니다. TypeScript 내부에 JSX가 포함되어 있어 단순한 템플릿 문자열로 작동하는 다른 프레임워크 보다 좋습니다.

아래 예제는 기본적인 React 앱 코드로 이러한 간단한 앱에서도 TypeScript를 사용하면 보다 뛰어난 개발자 경험을 가질 수 있습니다.

import { render } from 'react-dom'


const Hello = (props: { compiler: string; framework: string }) => {
  return (
    <div className="hello">
      <span>{props.compiler}</span> <span className="arrow">▸</span>
      <span>{props.framework}</span>
    </div>
  )
}


render(
  <Hello compiler="TypeScript" framework="React" />,
  document.getElementById('root')
)

예를 들어 활성 컴파일 시간 오류 분석을 사용하면 코드베이스에서 런타임 문제가 훨씬 줄어 듭니다.

  • 컴포넌트 이름을 잘못 입력하면 TypeScript에서 오류를 바로 표시합니다.
  • 컴포넌트 Props 속성 이름을 잘못 입력하면 즉각적으로 오류를 알려줍니다.
  • 필수 prop을 잊어버리고 입력하지 않으면 오류를 표시합니다.
  • 실수로 잘못된 속성 값 유형을 입력한 경우 오류를 발생시킵니다.

# TypeScript 프로젝트 생성

cra-template-typescript (opens new window) 템플릿을 사용해 손쉽게 TypeScript를 React에서 사용할 수 있습니다.

npx create-react-app <프로젝트> --template typescript
1

# 기존 프로젝트에 TypeScript 추가

CRA 툴체인을 사용해 생성한 프로젝트에 TypeScript를 추가해 사용하고자 한다면 다음 명령을 실행합니다.

npm i -D typescript @types/{node,react,react-dom,jest}
1

tsconfig.json (opens new window) 파일을 생성하고 다음 구성 코드를 작성합니다.

{
  "compilerOptions": {
    "target": "es5",
    "lib": [
      "dom",
      "dom.iterable",
      "esnext"
    ],
    "allowJs": true,
    "skipLibCheck": true,
    "esModuleInterop": true,
    "allowSyntheticDefaultImports": true,
    "forceConsistentCasingInFileNames": true,
    "noFallthroughCasesInSwitch": true,
    "module": "esnext",
    "moduleResolution": "node",
    "resolveJsonModule": true,
    "isolatedModules": true,
    "jsx": "react-jsx",
    "noEmit": true,
    "strict": true
  },
  "include": [ "src" ],
  "exclude": [ "node_modules" ]
}

# 확장자 설정

TypeScript를 사용해 React 앱을 개발할 경우, 확장자 사용은 다음 기준에 따라 작성합니다.

  • JSX를 사용할 경우, 확장자를 .tsx로 구성해야 합니다.
  • JSX를 사용하지 않을 경우, 확장자는 .ts로 구성할 수 있습니다.
