# React 개발 환경 구성

Webpack 모듈 번들러, Babel JavaScript 컴파일러 구성으로 React 앱을 개발하는 환경을 직접 구성해봅니다.

# 프로젝트 디렉토리

프로젝트 디렉토리를 생성한 후, 디렉토리로 이동합니다.

mkdir manual-webpack-babel-react && cd $_
1

# 프로젝트 초기화

프로젝트 초기화 후, public, src 디렉토리를 생성합니다.

npm init && mkdir public src
1

# 엔트리 파일(들)

정적 디렉토리의 엔트리 파일 index.html과 소스 디렉토리의 엔트리 파일 index.js 파일을 생성합니다.

code public/index.html src/index.js
1

public/index.html

정적 HTML 문서에 문서 제목, React 앱을 렌더링 할 루트 노드, 번들링 된 JavaScript 파일을 불러올 스크립트 코드를 추가합니다.







 



 
 




<!DOCTYPE html>
<html lang="ko-KR">
<head>
  <meta charset="UTF-8" />
  <meta http-equiv="X-UA-Compatible" content="IE=edge" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>React 앱 개발 환경 매뉴얼 구성</title>
</head>
<body>
  
  <div id="root"></div>
  <script src="js/main.js"></script>

</body>
</html>

# 구성 파일(들)

Webpack, Babel 구성 파일을 프로젝트 루트에 추가합니다.

code webpack.config.js .babelrc
1

webpack.config.js

Webpack 구성 파일에 엔트리, 아웃풋, 모듈(규칙: [...로더]), 타겟, 모드, 개발 서버 등 설정을 추가합니다.

const path = require('path')

module.exports = {
  entry: {
    main: './src/index.js'
  },
  output: {
    path: path.resolve(__dirname, 'public'),
    filename: 'js/[name].js'
  },
  module: {
    rules: [
      {
        test: /\.jsx?$/i,
        exclude: /node_modules/,
        use: 'babel-loader'
      }
    ]
  },
  target: 'web',
  mode: 'development',
  devServer: {
    contentBase: path.resolve(__dirname, 'public'),
    index: 'index.html',
    port: 9000,
    hot: true,
    compress: true,
    // writeDisk: true
  }
}

# CSS 스타일 로더

CSS 파일을 불러와 컴포넌트에 적용할 수 있도록 style-loader, css-loader 로더를 프로젝트에 설치합니다.

npm i -D style-loader css-loader
1

webpack.config.js

webpack 구성 파일의 모듈 규칙에 스타일 로더 코드를 추가합니다.

{
  module: {
    rules: [
      {
        test: /\.jsx?$/i,
        exclude: /node_modules/,
        use: 'babel-loader'
      },
      {
        test: /\.css$/i,
        exclude: /node_modules/,
        use: ['style-loader', 'css-loader']
      },
    ]
  },
}
